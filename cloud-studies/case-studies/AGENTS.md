# AGENTS.md

## 범위

- 이 규칙은 `cloud-studies/case-studies/` 아래의 사례 분석 문서에 적용합니다.

## 목적

- 실제 또는 가상의 클라우드 운영 사례를 문제, 원인, 대응, 재발 방지 중심으로 정리합니다.
- 특정 증상의 진단 절차 자체가 중심이면 `troubleshooting/`에 작성합니다.

## 작성 기준

- 사례의 배경과 영향 범위를 먼저 설명합니다.
- 타임라인, 관찰된 증상, 원인 후보, 최종 원인을 구분합니다.
- 사람, 조직, 고객을 식별할 수 있는 정보는 익명화합니다.
- 재발 방지는 실행 가능한 조치로 작성합니다.
- 근거 로그나 명령어가 있으면 민감 정보를 제거한 뒤 포함합니다.

## 권장 문서 구조

- `배경`: 시스템과 상황.
- `영향`: 사용자, 서비스, 비용, 보안 영향.
- `타임라인`: 주요 사건 순서.
- `원인 분석`: 후보와 배제 근거.
- `대응`: 즉시 조치와 복구 절차.
- `교훈`: 재발 방지와 운영 개선점.
