# AGENTS.md

## 범위

- 이 규칙은 `cloud-studies/experiments/` 아래의 실험, PoC, 재현 환경 문서에 적용합니다.

## 목적

- 설계나 운영 판단에 영향을 줄 수 있는 가정을 직접 검증합니다.
- 다른 사람이 같은 조건에서 재현할 수 있도록 환경과 절차를 남깁니다.

## 작성 기준

- 실험 목적과 검증하려는 가설을 먼저 작성합니다.
- OS, 런타임, 컨테이너 이미지, CLI 버전, 클라우드 리전 등 재현 조건을 기록합니다.
- 실행 명령어와 기대 결과, 실제 결과를 분리합니다.
- 실패한 실험도 삭제하지 말고 실패 조건과 배운 점을 남깁니다.
- 실제 클라우드 비용이 발생할 수 있으면 비용 위험과 정리 절차를 명시합니다.

## 권장 문서 구조

- `가설`: 확인하려는 주장.
- `환경`: 재현 조건과 버전.
- `절차`: 실행 순서와 명령어.
- `결과`: 관찰된 출력과 로그.
- `결론`: 가설 채택 여부와 후속 판단.
- `정리`: 생성한 리소스 삭제 방법.
