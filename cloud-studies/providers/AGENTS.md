# AGENTS.md

## 범위

- 이 규칙은 `cloud-studies/providers/` 아래의 공급자별 문서에 적용합니다.

## 목적

- AWS, GCP, Azure, Cloudflare 등 클라우드 공급자의 서비스 동작, 제약, 차이를 정리합니다.
- 공급자 고유 기능을 다른 공급자나 표준 개념과 비교할 수 있게 기록합니다.

## 작성 기준

- 공급자 이름, 서비스 이름, 리전, 확인 날짜를 명시합니다.
- 가격, 제한, 기본값처럼 변동 가능한 내용은 확인 날짜와 공식 링크를 함께 기록합니다.
- 공급자 문서에서 확인한 사실과 직접 실험한 결과를 구분합니다.
- 실험 절차가 길어지면 `experiments/`로 분리하고 이 문서에서는 요약과 링크만 둡니다.

## 권장 문서 구조

- `대상 서비스`: 공급자와 서비스 범위.
- `핵심 동작`: 중요한 기본값, 제약, 동작 방식.
- `다른 공급자와의 차이`: 비교 가능한 차이점.
- `운영 시 주의점`: 장애, 비용, 보안, 제한 관련 주의사항.
- `참고`: 공식 문서와 확인 날짜.
