# AGENTS.md

## 범위

- 이 규칙은 `cloud-studies/troubleshooting/` 아래의 장애 진단, 증상 분석, 재현 문서에 적용합니다.

## 목적

- 증상에서 출발해 원인 후보를 좁히는 과정을 재사용 가능한 형태로 기록합니다.
- DNS, 네트워크, IAM, 쿠버네티스, 데이터베이스, 관측성 등 문제 영역별로 정리합니다.

## 작성 기준

- 증상, 영향, 환경, 재현 절차, 관찰 결과, 결론을 분리합니다.
- 각 진단 단계는 왜 실행하는지와 어떤 결과를 기대하는지 함께 씁니다.
- 명령어 출력은 필요한 부분만 남기고 민감 정보는 제거합니다.
- 원인 후보를 배제한 근거를 남깁니다.
- 실험 환경을 포함하는 경우 정리 절차와 위험 요소를 기록합니다.

## 권장 문서 구조

- `증상`: 관찰된 문제.
- `환경`: OS, 런타임, 클라우드, 네트워크 조건.
- `재현`: 최소 재현 절차.
- `진단`: 실행한 명령어와 해석.
- `원인`: 확인된 원인과 배제한 후보.
- `해결`: 조치와 검증 방법.
- `주의`: 비슷한 증상과 구분할 점.
