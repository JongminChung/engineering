# AGENTS.md

## 범위

- 이 규칙은 `cloud-studies/` 전체에 적용합니다.
- 더 깊은 경로에 별도 `AGENTS.md`가 있으면 해당 규칙을 우선합니다.
- 이 공간은 클라우드 환경에서 학습한 개념, 공급자별 차이, 실험, 장애 분석, 참고 자료를 축적하는 용도입니다.

## 작성 원칙

- 문서는 한글로 작성합니다.
- 특정 클라우드 공급자에 종속된 내용은 공급자 이름을 명시합니다.
- 실험과 장애 분석은 재현 가능한 명령어, 환경, 관찰 결과, 결론을 분리해서 기록합니다.
- 추측과 확인된 사실을 구분합니다.
- 외부 문서를 근거로 삼은 경우 원문 링크와 확인 날짜를 남깁니다.
- 보안 정보, 계정 ID, 토큰, 내부 도메인, 실제 고객 데이터는 기록하지 않습니다.

## 디렉터리 역할

- `concepts/`: 클라우드 공통 개념과 동작 원리 정리.
- `providers/`: AWS, GCP, Azure, Cloudflare 등 공급자별 기능과 차이 정리.
- `experiments/`: 직접 실행한 검증, PoC, 재현 환경 정리.
- `case-studies/`: 실제 또는 가상의 사례를 문제, 원인, 대응, 교훈 중심으로 정리.
- `troubleshooting/`: 증상 기반 진단 절차와 장애 재현 기록.
- `references/`: 공식 문서, RFC, 외부 글, 도구 링크 모음.

## 문서 구조 권장

- 제목은 문제나 주제를 구체적으로 드러냅니다.
- 가능하면 `배경`, `환경`, `절차`, `결과`, `결론`, `참고` 순서를 사용합니다.
- 명령어는 fenced code block으로 작성하고 실행 위치나 전제 조건을 함께 적습니다.
- 결론은 재사용 가능한 판단 기준으로 정리합니다.
