# AGENTS.md

## 범위

- 이 규칙은 `cloud-studies/references/` 아래의 참고 자료 문서에 적용합니다.

## 목적

- 공식 문서, RFC, 블로그, 도구 문서, 발표 자료를 주제별로 모읍니다.
- 단순 링크 모음이 아니라 나중에 다시 찾을 수 있는 맥락을 함께 남깁니다.

## 작성 기준

- 링크 제목, URL, 확인 날짜, 핵심 내용을 함께 기록합니다.
- 공식 문서와 비공식 글을 구분합니다.
- 시간이 지나면 바뀔 수 있는 내용은 확인 날짜를 반드시 남깁니다.
- 실험이나 사례의 근거로 사용한 문서는 관련 문서에서 역링크를 남깁니다.

## 권장 문서 구조

- `공식 문서`: 공급자나 표준 문서.
- `표준/RFC`: 프로토콜과 표준 문서.
- `아티클`: 외부 해설 글.
- `도구`: CLI, 진단 도구, 실험 도구.
- `메모`: 해당 자료를 다시 볼 이유.
