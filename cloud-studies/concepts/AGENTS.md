# AGENTS.md

## 범위

- 이 규칙은 `cloud-studies/concepts/` 아래의 개념 정리 문서에 적용합니다.

## 목적

- 클라우드 공급자와 무관하게 재사용 가능한 개념, 프로토콜, 아키텍처 원리를 정리합니다.
- 공급자별 구현 차이는 핵심 개념을 설명하는 데 필요한 범위에서만 다룹니다.

## 작성 기준

- 용어 정의를 먼저 작성하고, 동작 원리와 예외를 뒤에 설명합니다.
- 개념 설명과 운영 경험을 섞지 말고, 운영 사례는 `case-studies/` 또는 `troubleshooting/`로 분리합니다.
- DNS, 네트워크, IAM, 스토리지, 쿠버네티스, 관측성처럼 주제별 하위 폴더를 사용할 수 있습니다.
- 특정 제품명을 다룰 때는 해당 개념의 일반 원리와 제품별 차이를 구분합니다.

## 권장 문서 구조

- `정의`: 용어와 범위.
- `동작 원리`: 주요 흐름과 구성 요소.
- `주의점`: 자주 오해하는 지점과 예외.
- `관련 주제`: 연결되는 개념 문서.
- `참고`: 공식 문서, RFC, 표준 링크.
