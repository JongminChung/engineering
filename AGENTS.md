# AGENTS.md

## 전역 커뮤니케이션 규칙

- 답변은 한글로 작성합니다.
- 내부 추론이나 고민은 영어로 수행하고, 결과만 한글로 전달합니다.
- 모호한 요청은 작업 전에 확인 질문을 합니다.
- 변경은 최소 단위로, 이유와 영향 범위를 함께 설명합니다.
- 설명에는 근거가 되는 파일/라인을 함께 언급합니다.
- 테스트/검증을 실행하지 못한 경우 사유와 대안을 함께 제시합니다.

## 작업 원칙

- 필요한 파일만 최소로 확인합니다.
- 기존 패턴/구조를 우선 존중합니다. 새 패턴 도입 시 이유를 간단히 설명합니다.
- 서브경로에 `AGENTS.md`가 있으면 해당 규칙이 우선합니다.
- 자동 생성/빌드 산출물은 사용자가 요청하지 않는 한 수정하지 않습니다.
- 검색은 `rg`를 우선 사용하고, 없으면 `find`로 대체합니다.

## 코딩/포맷 규칙

- 공통 스타일은 `.editorconfig`를 따릅니다.
- Java 포맷팅은 Spotless(Gradle) 기준을 따릅니다.
- TypeScript 포맷팅은 `biome.json` 기준을 따릅니다.
- 커밋 메시지는 Conventional Commits를 준수합니다.
- 보안 정보(키, 토큰, 비밀번호)는 절대 저장소에 남기지 않습니다.

## 테스트/검증 정책

- 변경 범위에 맞는 최소 테스트만 실행합니다.
- 시간이 오래 걸릴 경우 사전에 안내하고 동의를 구합니다.
- 테스트 실패 시 원인과 다음 조치 후보를 요약합니다.

## 기본 명령어

- `./gradlew build` — 전체 모듈 빌드 및 테스트 실행.
- `./gradlew test` — 전체 테스트 실행.
- `bun install` — 문서/포맷팅 도구 설치.
- `bun run check` — Biome 포맷/린트.
- `bun run check:markdown` — Markdown 자동 정리.
