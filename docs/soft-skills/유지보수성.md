# 유지보수성

- 모듈성
- 재사용성
- 분석성
- 수정 가능성
- 시험 가능성

## 유지보수성 관련 원칙들

- 깨진 창문이론
  - 나쁜 코드는 나쁜 코드를 유발
- 보이스카우트 규칙
  - "Always leave the campground cleaner than you found it"
- DRY(Don't repeat yourself)
  - 중복 코드를 작성하지 말자
- YAGNI(You aren't gonna need it)
  - 필요하기 전까지는 기능을 추가하지 않는다.
- KISS
  - "Keep it small and simple"
  - "Keep it short and simple"
  - "Keep it simple and stupid"

## 유지보수성을 높이는 방법

### 공통

- 개발 플랫폼 관례, 개발 커뮤니티 관례를 따른다
  - 하나의 스타일
  - 한 사람이 작성한 코드처럼 보이도록 한다
  - 본인의 취향이나 개성은 다른 곳에서
- 사용하지 않는 죽은 코드 제거
- 중복 코드 제거
- 매직 상수 남기지 않기
  - 의미를 나타내지 않는 문자, 숫자
- 관심사를 모듈로 분리한다.
- 코드 베이스를 작게 유지한다
