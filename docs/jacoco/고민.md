# 고민

## 대상

적용 대상의 범위가 너무 많다.

- gradle + spring boot application
- gradle + spring boot application (여러 개)
- gradle + spring boot application + common modules
- gradle + spring boot application (여러 개) + common modules(여러 개)

## 의문점

- GITLAB CI에서 작업과 무관한 테스트 커버리지 모두를 제공해야하는게 맞을까? (jacoco)

## 결론

각각의 모듈/서비스 별로 jacoco를 적용하여 테스트 커버리지를 제공하는 것이 좋을 것 같다.

모듈 A가 있다면 모듈 A는 단위 테스트를 표현하는 `test`, `integrationTest`로 구성된다.

모듈 A의 `test`는 단위 테스트를 표현하고, `integrationTest`는 모듈 A와 연관된 다른 모듈과의 통합 테스트를 표현한다.

해당 2개의 jacocoExec를 병합하고 하나의 jacocoReport를 생성하여 해당 모듈의 커버리지를 제공하는 태스크를 정의한다.

> 즉, 각 모듈/서비스별로 test + integrationTest exec를 병합해 단일 report 생성
